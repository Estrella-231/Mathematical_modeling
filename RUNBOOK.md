# 协作工作流操作手册（Runbook）

本手册用于具体执行与交接，建议每次任务开始前快速浏览。

## 1. 每日节奏

- **上午**：Gemini CLI 做长文解析和方案整理。
- **下午**：Claude Code 实现方案并生成实验结果。
- **晚上**：Codex Review + Debug，输出改动与风险说明。

## 2. 交付物清单（必须）

- `docs/00_problem_summary.md`
- `docs/01_assumptions.md`
- `docs/02_model_design.md`
- `docs/03_experiments.md`
- `docs/04_results.md`
- `docs/05_paper_outline.md`
- `docs/06_decision_log.md`

## 3. 交接模板

```
输入：
- 本次参考的文档：
- 本次新增假设：
- 本次改动的模型/参数：

输出：
- 关键结果：
- 生成的文件：
- 待验证事项：
```

## 4. 常见风险与对策

- **数据不足**：必须在 `docs/01_assumptions.md` 里明确假设。
- **模型过复杂**：优先采用可解释模型作为基线。
- **结果不可复现**：必须记录随机种子与配置。

## 5. Codex Review 最小要求

- 至少 3 个风险点或改进点
- 明确指出相关文件位置
- 提出修复建议或补丁

## 6. 论文一致性检查

- 论文中的指标是否与实验一致
- 图表标题与正文描述一致
- 所有假设在论文中有对应解释

---

执行过程中可根据需要扩展此手册，但请保持简洁、可复用。
